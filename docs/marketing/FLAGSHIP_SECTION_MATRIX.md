# Flagship Section Matrix

This matrix is the governing artifact for the canonical flagship route in
`apps/web/src/marketing/pages/landing/flagship/`.

Use it with:

- [MARKETING_FRONTEND_CONTRACT.md](../MARKETING_FRONTEND_CONTRACT.md)
- [FLAGSHIP_BASELINE.md](./FLAGSHIP_BASELINE.md)
- [VARIANT_TECHNIQUE_MATRIX.md](./VARIANT_TECHNIQUE_MATRIX.md)

## Rules

- Each section has one job only.
- Adjacent sections must not repeat both grammar and intensity.
- Only Hero and Canon may run at `High` intensity.
- `What to avoid` is a hard rejection list, not inspiration.
- No new shared marketing wrapper is allowed to satisfy this matrix.

## Locked Matrix

## Acceptance Gate

The flagship is ready only if:

- Hero has one dominant statement and one dominant proof object.
- Operating Laws reads as decompression, not as a second feature section.
- Benchmark reads as buyer criteria, not benefits marketing.
- Product Scope stays consequence-led, not taxonomy-led.
- Proof feels defensible, not merely premium.
- Canon remains the only cinematic chamber.
- Final CTA keeps `Enter Workspace` as the locked primary command.
