# Internationalization (i18n) — `apps/web` (SEA rollout)

**Status:** **Implemented** for the Vite SPA under **`apps/web`** using **i18next** + **react-i18next** + **i18next-browser-languagedetector**. Canonical authoring locale is **`en`**; active locales are **`en`**, **`ms`**, **`id`**, **`vi`**.

**Source of truth layout:** [`apps/web/src/app/_platform/i18n/`](../../apps/web/src/app/_platform/i18n/) — runtime bootstrap (`adapters/i18next-adapter.ts`, `policy/i18n-policy.ts`, `services/i18n-format-service.ts`), per-locale JSON under `locales/<lng>/<namespace>.json`, glossary (`glossary/`), and audit outputs (`audit/`).

---

## Runtime behavior

| Topic                 | Implementation                                                                                                                                                                                                                              |
| --------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Bootstrap**         | `initI18n()` in [`main.tsx`](../../apps/web/src/main.tsx) **before** `createRoot` renders (static bundled resources).                                                                                                                       |
| **Default namespace** | `shell` (`defaultNS`); registered namespaces: `shell`, `auth`, `dashboard`, `invoice`, `allocation`, `settlement`.                                                                                                                          |
| **Detection order**   | **Development:** `localStorage` → `?lng=` query → `navigator` → `htmlTag`. **Production:** `localStorage` → `navigator` → `htmlTag` (no querystring override).                                                                              |
| **Persistence**       | `localStorage` key **`afenda.locale`** (see [`i18n-policy.ts`](../../apps/web/src/app/_platform/i18n/policy/i18n-policy.ts)).                                                                                                               |
| **Fallback**          | `fallbackLng: 'en'`, `load: 'languageOnly'`, `supportedLngs` locked in policy.                                                                                                                                                              |
| **React**             | `initReactI18next`, `interpolation.escapeValue: false`, `react.useSuspense: false`.                                                                                                                                                         |
| **Formatting**        | Shared helpers only: **`formatCurrency`**, **`formatDate`**, **`formatNumber`** in [`i18n-format-service.ts`](../../apps/web/src/app/_platform/i18n/services/i18n-format-service.ts) (aligned to active i18n language → BCP 47 for `Intl`). |
| **Missing keys**      | Logged in **development** via `missingKeyHandler`; production must not surface raw implementation details to users.                                                                                                                         |

---

## Keys, namespaces, and governance

- **Canonical keys** live in English JSON only; other locales must mirror the same key paths (validated by **`pnpm run script:validate-i18n-policy`**).
- **Release namespaces** (stricter CI): `shell`, `auth`, `dashboard` (see `RELEASE_NAMESPACES` in policy).
- **Glossary & audit:** [`glossary/canonical-terms.json`](../../apps/web/src/app/_platform/i18n/glossary/canonical-terms.json) + [`glossary/decisions.json`](../../apps/web/src/app/_platform/i18n/glossary/decisions.json); audit outputs [`audit/conflicts.json`](../../apps/web/src/app/_platform/i18n/audit/conflicts.json), [`audit/provenance.json`](../../apps/web/src/app/_platform/i18n/audit/provenance.json). Regenerate conflicts with **`pnpm run script:generate-i18n-crossref-audit`** (sample corpora under [`apps/web/scripts/i18n/data/`](../../apps/web/scripts/i18n/data/)).
