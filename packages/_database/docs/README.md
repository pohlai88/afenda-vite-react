# `@afenda/database` docs

This documentation set was intentionally reset after the database restart.

The old docs mixed:

- current package behavior
- removed schema surfaces
- future-state ERP design
- raw source/reference material

That made the folder unreliable as a source of truth.

## Current state

The package is operating from a minimal foundation. Documentation will be rebuilt from that reality first, then expanded deliberately.

See:

- [Clean Rewrite Proposal](./CLEAN_REWRITE_PROPOSAL.md)

## Documentation rule

No document in this folder should describe a schema, service, or workflow as current unless it exists in the active package.
