# Feature Priority Matrix

## Critical

Build or adopt first when the candidate protects company access, money, data, auditability, compliance, or operations.

Examples:

- IAM / SSO
- Secrets management
- Monitoring / uptime
- Backup / recovery
- Document management
- Internal CRM / support
- BI / reporting
- Audit / compliance evidence

## Essential

Build second when the candidate improves repeated team workflows.

Examples:

- AI work assistant
- Knowledge base
- Workflow automation
- Project management
- Forms / surveys
- Product analytics
- Data integration
- Translation / localization

## Good-To-Have

Build later when the candidate is useful but not operationally urgent.

Examples:

- Website builder
- Screen recording
- Bookmark manager
- Community tools
- Design / prototyping
- Personal productivity utilities
- Experimental AI agents
