# Rules

This directory holds durable repository governance that is stricter than prose-only documentation.

Use `rules/` for:

- machine-readable policy inputs
- refactor planning surfaces tied to governance
- stable conventions that should survive feature churn

Do not use `rules/` for:

- feature implementation notes
- temporary experiments
- duplicated copies of docs already governed elsewhere

Current governed areas:

- [`filesystem-governance`](./filesystem-governance/README.md)
